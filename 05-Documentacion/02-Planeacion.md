# Planeación del Proyecto

## Objetivo General
Diseñar e implementar un sistema de botón de pánico que permita a usuarios generar alertas inmediatas a entidades de seguridad en situaciones de riesgo.

## Objetivos específicos
- Analizar la problemática de seguridad y los requerimientos de los usuarios.
- Diseñar la arquitectura tecnológica del sistema (aplicación + servidor + base de datos).
- Desarrollar un prototipo funcional usando metodologías ágiles (Scrum).
- Evaluar riesgos y proponer mitigaciones.
- Validar la aceptación del sistema con los stakeholders.

## Alcance
- **Corto plazo**: Desarrollo del prototipo mínimo viable (MVP) con app móvil y servidor.
- **Mediano plazo**: Integración con sistemas de seguridad locales y pruebas en campo.
- **Largo plazo**: Escalamiento a nivel regional con integración de múltiples instituciones.

### Restricciones
- Limitación de presupuesto inicial.
- Dependencia de internet en algunos entornos.
- Capacitación básica para usuarios.

### Criterios de aceptación
- El botón envía una alerta en menos de 3 segundos.
- El sistema genera reportes de incidentes.
- Interfaz amigable para usuarios no técnicos.

## Metodología seleccionada
Se empleará **Scrum**, que permite un desarrollo ágil mediante sprints cortos y retroalimentación constante.  
Herramientas:
- Trello (gestión de tareas).
- Miro (mapas de empatía y diagramas).
- GitHub (gestión de código y documentación).

## Matriz de riesgos

| Riesgo | Probabilidad | Impacto | Estrategia |
|--------|--------------|---------|------------|
| Fallo en la red de internet | Alta | Alto | Implementar redundancia y alertas offline |
| Baja adopción del sistema | Media | Alto | Campaña de capacitación y difusión |
| Costos de hardware | Alta | Medio | Buscar proveedores y convenios |
| Vulnerabilidad de datos | Media | Alto | Cifrado y protocolos de seguridad |
