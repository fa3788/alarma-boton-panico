# Product Backlog

1. **Investigación inicial** – Identificación de requerimientos.
2. **Diseño de prototipo** – Interfaz básica del botón de pánico.
3. **Desarrollo del backend** – Servidor para recibir alertas.
4. **Integración con base de datos** – Registro de eventos.
5. **Desarrollo de frontend** – Aplicación o panel de usuario.
6. **Pruebas de sistema** – Validación de envío de alertas.
7. **Documentación final** – Manual de uso y entrega de resultados.
